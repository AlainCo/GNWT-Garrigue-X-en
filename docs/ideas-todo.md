## Status Vocabulary

Use the following markers when they help clarify priority or outcome:

- `[URGENT]`: must be addressed quickly as it blocks quality, reliability, or useful progress

- `[SOON]`: good candidate for short-term implementation

- `[LATER]`: worth keeping, but intentionally postponed

- `[TODISCUSS]`: requires a design discussion before implementation

- `[DONE]`: processed and temporarily kept for traceability before cleanup

- `[ABANDONED]`: explicitly rejected, temporarily kept to avoid the same idea being reopened by mistake

## Backlog Hygiene

- when an item is addressed, either delete it or move it to a short history section instead of leaving it mixed with active items

- when an item is no longer relevant, delete it rather than keeping obsolete noise in the backlog

- regularly group similar items and merge duplicates when they describe the same future evolution
